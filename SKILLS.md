# Skills

## Overview
TriageAcceleration is an AI-powered hospital management system built for the BC Hackathon (Track 1: Biology and Physical Health). It accelerates the triaging process, optimizes staff delegation, and reduces ER wait times across understaffed Canadian hospitals.

## Technical Skills

### Frontend
- **Next.js** — React-based framework for the patient and staff web app
- **Tailwind CSS** — Utility-first CSS for rapid UI development

### Backend
- **Python** — Core backend language
- **FastAPI** — High-performance API framework for AI and data endpoints
- **Flask** — Localhost hosting and lightweight routing
- **SQLite3** — Local database for patient records and staff data

### AI / Machine Learning
- **Natural Language Processing** — Parsing patient symptoms into hospital-standard terminology
- **Severity Classification** — AI-driven urgency ranking based on symptoms, age, and medical history
- **Patient-Staff Matching** — Matching patients to departments and specialists based on symptoms and staff specialization
- **Medical Document Summarization** — Condensing patient history into scannable summaries per department
- **Task Scheduling** — AI-assisted patient queue optimization by urgency and department load

### Data
- **Synthetic Medical Data** — Generated datasets for development and testing
- **Patient Data Model:**
  - Patient ID (unique identifier)
  - Full Name
  - Gender
  - Health Number
  - Age
  - Symptoms / Medical History

### Tools & Platforms
- **Claude Code MCP** — AI-assisted development environment
- **Git & GitHub** — Version control and collaboration
- **Localhost Development** — Local-first architecture for hospital data privacy

## Domain Skills
- **Emergency Triage Protocols** — Understanding CTAS (Canadian Triage and Acuity Scale) severity levels
- **Hospital Resource Management** — Tracking department busyness and staff availability
- **Patient Flow Optimization** — Reducing bottlenecks from ER intake to specialist delegation
- **Medical Data Privacy** — Handling sensitive patient records with local storage
- **Clinical Documentation** — Translating patient-submitted data into hospital-standard language

## Key Product Features
1. **Patient Self-Check-In** — Patients submit symptoms and medical data via web app on arrival
2. **AI Triage Assistance** — Symptoms parsed into hospital lingo with severity ranking for triage staff
3. **Smart Patient Delegation** — AI matches patients to departments/doctors based on symptoms, specialization, and department load
4. **Live Wait Updates** — Patients receive real-time queue updates on the same web app
5. **Unified Medical Summary** — AI compiles patient info into a single scannable document per department
6. **Staff Dashboard** — Staff submit department, specialization; system tracks availability
7. **Medical History Portal** — Patients can submit/manage their medical history for appointments
